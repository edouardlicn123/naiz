# 测试流程规范

## 1. 概述

Naiz 引擎的测试全流程为：**编译 → 注入 → 模拟器运行 → 日志提取 → 验证**。
本文档定义各环节的标准操作和验证方法。

### 数据流

```
projects/<name>/   core/（HAL/引擎库）
       │               │
       ▼               ▼
 ia16-elf-gcc      ia16-elf-ld
       │               │
       └───────┬───────┘
               ▼
        games/<name>/engine.exe
               │
               ▼ inject.py
        disks/<name>.hdi
               │
               ▼ NP2kai (sdlnp21kai_sdl2)
        ENGINE.LOG（在 DOS 内生成）
               │
               ▼ inject.py --extract
        logs/ENGINE.RUN.<timestamp>.log
```

---

## 2. 测试级别

### 2.1 单元测试（宿主机）

引擎代码使用 gcc-ia16 交叉编译，宿主机上无法直接运行。
工具链（`tools/naiz_img/`）的单元测试通过 Python 直接执行：

```bash
# 验证 FAT 解析
python3 -c "
from naiz_img import open_image
from naiz_img.fat import NAIZFatFS
img = open_image('tools/msdos5.hdi')
fs = NAIZFatFS(img)
assert fs.fat_type in (12, 16)
print('FAT fs OK:', fs.fat_type, 'clusters')
"

# 遍历文件验证
python3 -c "
from naiz_img import open_image
from naiz_img.fat import NAIZFatFS
img = open_image('tools/msdos5.hdi')
fs = NAIZFatFS(img)
for path, e in fs.walk():
    if path.upper() == '/IO.SYS':
        print('IO.SYS found')
        break
"
```

### 2.2 冒烟测试（engine.exe）

验证引擎能在 DOS 下启动并正确返回：

```bash
# 使用 test_hang.c（最小退出测试）
cd projects/demo-A1 && make clean && make

# 检查 MZ 头
python3 -c "
import struct
with open('games/demo-A1/engine.exe','rb') as f:
    e=f.read()
cs = struct.unpack_from('<H',e,0x16)[0]
ip = struct.unpack_from('<H',e,0x14)[0]
ss = struct.unpack_from('<H',e,0x0E)[0]
sp = struct.unpack_from('<H',e,0x10)[0]
print(f'CS={cs} IP={ip} SS={ss} SP={sp}')
assert cs == 0 and ip == 0, 'MZ entry point must be CS=0 IP=0'
"
```

### 2.3 集成测试（NP2kai）

完整端到端测试：编译 → 注入 → 模拟器运行 → 提取日志。

#### 标准流程

```bash
# 1. 编译
make -C core clean all
cp core/engine.exe games/demo-A1/

# 2. 注入 HDI
python3 -m tools.naiz_img.inject --game demo-A1 --yes

# 3. 运行 NP2kai（无头 + 超时）
echo | SDL_VIDEO_WINDOW_POS=9999,9999 SDL_AUDIODRIVER=dummy \
  timeout 35 python3 -c "
import sys; sys.path.insert(0, 'tools')
from env_setup.install_env import cmd_test_hdi
cmd_test_hdi(hdi_path='disks/demo-A1.hdi', emulator='ia32')
"

# 4. 提取引擎日志
python3 -m tools.naiz_img.inject --game demo-A1 --extract DEMO-A1/ENGINE.LOG

# 5. 查看
cat logs/ENGINE.RUN.*.log
```

#### 通过标准

| 检查点 | 方法 | 预期 |
|--------|------|------|
| MZ 入口正确 | objdump / python | CS=0, IP=0 |
| NP2kai 不崩溃 | 无 `interrupt: num = 0x06` | 模拟器正常运行 |
| 引擎自动退出 | 超时 35s 内完成 | 返回 DOS 提示符 |
| 日志生成 | `--extract` 有输出 | `ENGINE.RUN.*.log` 存在 |
| 日志内容 | `cat` 查看 | 含 `OPEN`、`DONE` 等标记 |

## 4. 测试工具

### test_hdi.sh

```bash
./test_hdi.sh                    # 交互选择 HDI 和核心
./test_hdi.sh demo-A1            # 直接运行 demo-A1
./test_hdi.sh demo-A1 ia32       # 指定核心
./test_hdi.sh -h                 # 帮助
```

内部调用 `tools/env_setup/install_env.py` 的 `cmd_test_hdi()`：

1. 列出 `disks/*.hdi` 供选择
2. 生成临时 `np2kai.cfg`（HDD1FILE 指向所选 HDI）
3. 设置 SDL 环境变量
4. 等待 Enter 后启动 NP2kai
5. 记录日志到 `logs/test_<game>_<timestamp>.log`

### inject.py --extract

提取引擎日志到宿主机：

```bash
python3 -m tools.naiz_img.inject --game demo-A1 --extract DEMO-A1/ENGINE.LOG
# 输出: logs/ENGINE.RUN.20260526.143021.log
```

## 5. 常见问题诊断

| 症状 | 可能原因 | 排查方法 |
|------|----------|----------|
| UD 异常 (int 0x06) | MZ 入口错误 | 检查 `CS=0, IP=0`, `*crt0.o(.text)` 在链接脚本最前 |
| 黑屏无反应 | BIOS/核心不匹配 | 确认使用 IA32 核心 + PC-9821 BIOS |
| 键盘卡死 | Wayland 输入抓取 | `SDL_VIDEODRIVER=x11` |
| 日志为空 | 未调用 `log_close()` | 确认退出前 flush + close |
| 无 C: 盘 | SASI 未启用 | 确认 `SUPPORT_SASI` 已定义 |
| HDI 无法启动 | IO.SYS 不在根目录第一 | 用 `--list-files` 检查 |

## 6. 快速参考

```bash
# 完整测试循环
cd ~/naiz/projects/demo-A1 && make clean && make && \
python3 -m tools.naiz_img.inject --game demo-A1 --yes && \
../test_hdi.sh demo-A1

# 仅编译引擎
make -C core clean all

# 仅注入（引擎未变时）
python3 -m tools.naiz_img.inject --game demo-A1 -y

# 仅启动模拟器
../test_hdi.sh demo-A1

# 提取日志
python3 -m tools.naiz_img.inject --game demo-A1 --extract DEMO-A1/ENGINE.LOG

# 检查 MZ 头
ia16-elf-objdump -d -m i8086 games/demo-A1/engine.exe | head -10
```
