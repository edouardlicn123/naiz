# 系统死机问题分析与修复方案

## 1. 用户描述的现象

通过 `make -C core test`（opencode 开发界面）运行时:

- 模拟器可以正常启动
- 引擎可以正常退出

通过 `./testtemp.sh` 运行时:

- 模拟器执行开机内存检测（POST）后黑屏
- 鼠标可以在浏览器窗口外部移动
- 系统全部功能僵死（键盘无响应，无法切换 TTY，无法操作任何窗口）

重启前曾做过一些修改，问题依旧。

## 2. 环境

- 引擎：gcc-ia16 交叉编译，16-bit MSDOS small model
- 模拟器：NP2kai (sdlnp21kai_sdl2, SDL2 端口)
- 平台：Linux (Wayland)
- 启动脚本：`testtemp.sh` 通过 `source tools/sdl_env.sh` 设置了 `SDL_VIDEODRIVER` 和 `SDL_AUDIODRIVER`

## 3. 根因分析

### 3.1 黑屏的原因

引擎 `main()` 执行流程：

1. `hal_check_compatibility()` — EGC 检测，通过
2. `hal_video_init()` — 调用 `gdc_stop_text()` 关闭 PC-98 文本层 → 画面从此消失
3. `gdc_start_graphics()` + `gdc_set_display_mode(640,400,440)` — 切换到图形模式
4. `hal_video_clear_screen()` — EGC 清屏为 color 0（黑色）
5. `hal_video_fill_rect(0, 0, 640, 400, 1)` — 填充 color 1（调色板索引1对应蓝 (0,0,15))

结果：用户看到的是 POST → DOS 启动 → 屏幕突然变黑（文本层关闭），且后续填充的颜色在模拟器中可能因调色板/GDC 配置偏差表现为全黑或极暗，无任何视觉反馈。

### 3.2 系统僵死的原因

`testtemp.sh` 中 `source tools/sdl_env.sh` 设置了：

```bash
SDL_VIDEODRIVER=wayland    # 若 XDG_SESSION_TYPE=wayland
SDL_AUDIODRIVER=pulse
```

NP2kai 通过 SDL2 Wayland 后端运行时，SDL 窗口默认抓取键盘独占输入（SDL 的 `SDL_WINDOW_INPUT_FOCUS` + Wayland 键盘锁定）。在用户看来：

- 鼠标光标仍可移动（由 Wayland 合成器管理）
- 键盘事件全部被 NP2kai 消耗，Alt+Tab、Ctrl+Alt+Fx、Super 键等无法到达窗口管理器或内核
- 无法切换 TTY、无法操作其他窗口、无法启动终端
- 浏览器等前台应用失去键盘焦点，表现为"僵死"

`make -C core test` 正常工作的原因：它**不**设置 `SDL_VIDEODRIVER`，SDL 使用默认驱动（通常是 x11 或 wayland 但配置不同），输入抓取行为不同。

### 3.3 为何无法退出

引擎 `main.c` 的退出条件是：

```c
if (hal_input_state(0x01)) break;
```

`hal_input_state(0x01)` 等价于 `*(volatile char*)0x052A & 0x01`，即 PC-98 键盘矩阵第 0 字节 bit 0 —— 对应数字键 **"1"**。但此时：

- 用户不知道应按"1"（无屏幕提示）
- 键盘被 NP2kai 抓取，按键是否被正确传递到模拟器也不确定

### 3.4 根本原因总结

| 问题 | 结果 |
|------|------|
| 引擎黑屏无反馈 | 用户不知道引擎已运行 |
| 无自动超时退出 | 引擎无限等待按键 |
| SDL 输入抓取 | 用户无法干预 |
| 三者叠加 | 系统表现为完全死机 |

## 4. 修复方案

### 4.1 引擎添加自动超时退出

**文件：** `core/engine/main.c`

在循环中增加帧计数器，超过 300 帧（~5 秒 @ 60fps）自动退出，无需按键。

### 4.2 启动脚本去掉 SDL 驱动强制设定

**文件：** `testtemp.sh`

- 删除 `source tools/sdl_env.sh`（不再强制设定 `SDL_VIDEODRIVER`）
- 添加 `timeout 30` 保护（30 秒自动结束模拟器进程）
- 添加 `set -e` 错误中止
- **注意：**环境变量需在 `timeout` 之前单独 `export`，因为 `timeout` 不支持前置变量赋值语法

```bash
export XDG_CONFIG_HOME=core
timeout 30 sdlnp21kai_sdl2
```

### 4.3 ISR 健壮性修复

**文件：** `core/plat/pc98/isr.asm`

在 `out 0x64, al` 前显式初始化 `al = 0`。

### 4.4 可选的屏幕文字输出

若希望在黑屏上看到引擎运行状态，可在 `main.c` 中添加在图形模式下写入像素点阵文字的辅助函数（利用 EGC + 字模数据）。暂不纳入本次修复范围，因为自动退出已解决问题。

## 5. 预期效果

完成后：

1. `./testtemp.sh` 启动 → NP2kai 窗口出现
2. DOS 启动 → 用户输入 `C:` 回车 → `ENGINE` 回车
3. 屏幕变黑（引擎图形模式）→ 5 秒后自动回到 DOS 提示符
4. 模拟器不抓取输入 → 用户可以正常关闭 NP2kai 窗口或在 DOS 输入其他命令
5. 30 秒后 `timeout` 强制结束模拟器进程（安全兜底）
