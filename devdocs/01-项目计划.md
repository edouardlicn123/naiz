# 新建 PC-98 Galgame 引擎 — 项目计划

## 概述

本项目旨在以 MHVNVisualNovelEngine 为核心参考，吸收 master.lib、ReC98、pmdmini、xsystem35-sdl2 等开源项目的思想和经验，全新书写一个面向 PC-98 实模式环境的 Galgame 引擎，并配套 Python 工具链。

## 核心设计

### 硬件抽象层 (HAL)

`core/plat/hal.h` 定义引擎与平台的全部边界，引擎核心只通过 hal.h 与平台交互：

| HAL 接口 | PC-98 后端 | SDL2 后端（未来） |
|----------|-------------|------------------|
| hal_video_init/blit/palette/vsync | EGC/GRCG/GDC | SDL2 surface + texture |
| hal_input_poll/state | BIOS 键盘区 | SDL2 event |
| hal_audio_play_music/stop/sfx | PMD.COM + 蜂鸣器 | pmdmini + SDL_mixer |
| hal_file_open/read/write/close/seek | DOS INT 21h | stdio/fopen |

### 场景脚本系统

场景脚本采用**文本源码 → 编译器 → 字节码**模式：

1. **源码格式**：类 NScripter 风格的文本脚本（`.mhn` 文件）
   ```
   @scene 1
   @bg 1
   @char 0 "主角"
   @text "你好，世界！"
   @choice "去商店" → scene 2
   @choice "回家"   → scene 3
   ```

2. **Python 编译器**（`tools/script_compiler/`）：将 `.mhn` 编译为二进制字节码

3. **引擎 VM**（`core/engine/scene_vm.c`）：参考 MHVN98 的 ~50 opcode，解释执行字节码

## 构建系统（Build System）

### 工具链

| 组件 | 用途 |
|------|------|
| `ia16-elf-gcc` | C 交叉编译器（IA-16, 16-bit real mode） |
| `ia16-elf-as` | GNU assembler for IA-16 |
| `ia16-elf-ld` | 链接器 |
| `make` | 构建自动化 |

### 编译参数

编译使用 gcc-ia16 的 small model，链接时通过 `-Wl,-melf_i386_msdos_mz` 生成 MZ 格式 .EXE，不加手写链接脚本（gcc-ia16 自带 small model linker）。

```makefile
# Makefile（参考 MHVN98 构建方式）
CC       = ia16-elf-gcc
AS       = ia16-elf-as
CFLAGS   = -march=i386 -mcmodel=small -Os -ffreestanding -nostdlib \
           -Iengine -Iplat -I.
LDFLAGS  = -Wl,-melf_i386_msdos_mz,-M,-static,--relax,-n,--no-dynamic-linker,--print-memory-usage
LDLIBS   = -li86
OUTPUT   = engine.exe
```

`-li86` 提供 `main()` 入口点和标准 C 库，无需手写入口或链接脚本。

## 约束规则

1. **ref_projects/ 只读** — 不在此目录下写任何代码。每个子项目有 `README.md` 注明名称、GitHub URL、许可证。
2. **core/ 和 tools/ 是引擎本体** — 代码放在这里，不直接 `#include` 或 import ref_projects 中的文件。
3. **使用参考代码的两种方式**：
   - **优先**：学习思路后独自重写
   - **备选**：复制文件到 core/ 或 tools/，但文件开头必须加注释块注明来源项目名称、GitHub URL、许可证
4. **GPL 传染性注意** — GPL v2 代码复制后整个引擎可能受 GPL 约束，优先借鉴而非复制。
5. **projects/xxx/ 是游戏项目** — 每个项目独立，有自己的素材和构建。
6. **HAL 是唯一边界** — 引擎核心代码不得直接引用 PC-98 硬件操作。
