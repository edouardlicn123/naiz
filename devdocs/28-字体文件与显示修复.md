# 28 — 字体文件生成与显示效果修复

## 问题

1. **FONT.DAT 缺失**：`games/demo-A1/` 下无 FONT.DAT，引擎 `init_font_file()` 静默跳过，使用硬编码 `ascii_char_cache`
2. **Line double bug**：`main.c:31` 用 `|` 合并 `gdc_set_mode1()` 参数，导致 `GDC_MODE1_LINEDOUBLE_OFF` 被 OR 掉，PC-98 以 200 行显示 400 行内容（每行加倍），产生"垂直镜像"假象
3. **调试路径残留**：`textengine.c:78` 的 `write_string(root_info.cur_text_path, 0, 0, ...)` 把 "EN_GB.TXA" 打在屏幕左上角
4. **font_path 未修补**：`games/demo-A1/ROOTINFO.DAT` 的 font_path 偏移 +0x16 处为垃圾字节；`inject_common.py:276` 虽在注入时自动修补，但 demo 的根目录来源文件未修补

## 方案

### Step 1 — 生成 FONT.DAT
使用 `tools/gen_font.py`，从 `ascii_char_cache`（`core/data/font_ascii.c`）提取 94 个 8×16 点阵字符，打包为 FONT.DAT：
- RangeList: 512 字节（1 个 range，起始 0x21，94 字符）
- CharInfo 数组: 94 × 4 字节
- Glyph 数据: 94 × 16 字节

### Step 2 — 修补 ROOTINFO.DAT font_path
将 `games/demo-A1/ROOTINFO.DAT` +0x16 的 12 字节写为 `FONT.DAT\0\0\0\0`

### Step 3 — 修复 main.c line double bug
将：
```c
gdc_set_mode1(GDC_MODE1_COLOUR | GDC_MODE1_LINEDOUBLE_OFF | GDC_MODE1_DISPLAY_ON);
```
改为三次分别调用：
```c
gdc_set_mode1(GDC_MODE1_COLOUR);
gdc_set_mode1(GDC_MODE1_LINEDOUBLE_OFF);
gdc_set_mode1(GDC_MODE1_DISPLAY_ON);
```

### Step 4 — 删除 textengine.c 调试输出
删除 `textengine.c:78` 的 `write_string()` 调用

### Step 5 — 编译、注入、测试
```bash
make && makegame.sh make demo-A1 && makegame.sh test demo-A1
```
