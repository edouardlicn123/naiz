1. **获取 NP2kai 对配置键名的官方说明**
   - 运行 `sdlnp21kai_sdl2 --help` 或查看 `/usr/share/doc/sdlnp21kai` 指令集，寻找配置语法。
   - 列出所有可用键与呈现为 `key=value` 的形式。

2. **备份现有配置**
   ```bash
   cp core/sdlnp21kai/np21kai.cfg core/sdlnp21kai/np21kai.cfg.bak
   ```

3. **根据官方键名重新编写最小配置**
   *示例（待核对官方键名后更新）*
   ```ini
   [NP2]
   biospath = /home/edo/naiz/tools/pc98bios/ROM.ROM
   HDD1FILE = /home/edo/naiz/tools/DOS62.hdi
   boot_hd   = 1
   boot      = hd
   ```

4. **验证 48 MB HDI 是否正确**
   ```bash
   python3 - <<'PY'
import os
path='tools/DOS62.hdi'
print('size', os.path.getsize(path))
print('first sector', open(path,'rb').read(512)[:64])
PY
   ```

5. **测试 NP2kai（带最小配置）**
   ```bash
   cd /home/edo/naiz
   export XDG_CONFIG_HOME="$PWD/core"
   export SDL_VIDEO_ALLOW_SCREENSAVER=1
   nice -n 10 timeout 15 sdlnp21kai_sdl2 > logs/debug.log 2>&1
   ```
   - 查看 `logs/debug.log` 里是否出现 DOS 提示符 `A:\>`。

6. **如无 DOS，开启 NP2kai 详细日志**（若支持）
   ```bash
   sdlnp21kai_sdl2 --config core/sdlnp21kai/np21kai.cfg -V > logs/verbose.log 2>&1
   ```

7. **使用 QEMU 交叉验证**
   ```bash
   qemu-system-i386 -boot c -hda tools/DOS62.hdi -L tools/pc98bios
   ```
   - 若 QEMU 能成功启动 DOS，则问题在 NP2kai 配置；若 QEMU 也失败，检查 ISO/BIOS 与 HDI。

8. **若 QEMU 成功，迁移 QEMU 配置至 NP2kai**
   - 记录 QEMU 启动时使用的 BIOS 路径、磁盘映像，按 NP2kai 官方键名更新 `np21kai.cfg`。

9. **记录最终可用配置**
   - 把工作中的 `np21kai.cfg` 复制到 `devdocs/`，附上对应 `logs/debug.log` 与 `logs/verbose.log`。<br>
   - 说明最终成功的键名及其值，便于后续维护。

10. **若仍有问题**
   - 把最新的 `logs/debug.log` 与 `logs/verbose.log` 交给我，我将进一步定位。
