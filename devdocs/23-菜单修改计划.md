# 菜单与脚本重构方案

## 目标

1. `env_menu` 从 `start.sh` 移入 `install_env.py`，变成交互模式
2. 合并 `make_hdi.sh` + `test_hdi.sh` + 编译项目为 `makegame.sh`
3. `start.sh` 瘦身

## 最终结构

```
项目根目录
├── start.sh                    # 仅 3 项：1.查看状态  3.更新子项目  4.编译引擎  0.退出
├── tools/
│   ├── env_setup/
│   │   └── install_env.py      # 无参 → 交互配置菜单（原 env_menu 7 项）
│   │                           # 有参 → 子命令模式（不变）
│   ├── makegame.sh             # make/test/build 三子命令 + 无参交互
│   └── naiz_img/
│       ├── inject.py           # 不变（makegame.sh make 的后端）
│       ├── inject_common.py    # [新建] 增量注入核心
│       └── inject_file.py      # 不变
```

**不再有** `make_hdi.sh`、`test_hdi.sh`

各脚本独立运行，互不嵌套：

```bash
start.sh                              # 启动主菜单
python tools/env_setup/install_env.py # 安装/配置开发环境
makegame.sh make demo-A1              # 游戏制作/测试/编译
makegame.sh                           # 交互式游戏工作流
```

### 2.3 镜像源处理

选项 7 的交互逻辑（选 github 还是 china 镜像）目前是 `start.sh` 内联的 bash，搬进 Python 后变成 `cmd_interactive()` 的一部分。内部调用时需要通过某种机制设置 MIRROR——最简单的方式是在交互函数内直接调用 `_mirror_init()` 的写入逻辑：

```python
elif choice == "7":
    print("\n───── Git 仓库来源 ─────")
    print("  1) GitHub（海外直连）")
    print("  2) 国内镜像（Gitee/GitCode，中国大陆加速）")
    m = input("请选择 [1/2]: ").strip()
    if m == "1":
        _set_mirror("github")
    elif m == "2":
        _set_mirror("china")
    input("按 Enter 键继续...")
```

新增辅助函数：
```python
def _set_mirror(mirror_name):
    global MIRROR
    MIRROR = mirror_name
    cfg = _read_conf()
    cfg["mirror"] = MIRROR
    _write_conf(cfg)
    print(f"  已设置镜像源: {mirror_name}")
```

保持与现有 `--mirror` CLI 参数一致的行为。

### 2.4 sudo 保持

原 `start.sh` 的 `env_menu` 中，选项 2/3/4/5 有 sudo 保持机制（`sudo -v` + background `sudo -n true`loop）。搬进 Python 后：

- `cmd_system_tools()`、`cmd_np2kai()`、`cmd_backup_emu()`、`cmd_build_i286()` 内部已有 `_sudo_init()` 调用，会在开始前缓存 sudo 凭证
- 不需要在交互菜单层再做 sudo 保持
- 交互菜单中的选项 2/3/4/5 直接调对应 `cmd_*()` 即可

### 2.5 `_sudo_init()` 改进

当前 `_sudo_init()` 只调一次 `sudo -v`。对于需要 sudo 的长时间操作（如 `apt-get install`），一次 `sudo -v` 的缓存时间可能不够。建议保持现有实现不变（sudo 默认超时 15 分钟，足够编译）。

## 3. 新建 `tools/makegame.sh`

### 3.1 接口

| 命令 | 作用 | 后端 |
|------|------|------|
| `makegame.sh make <game>` | 制作 HDI | `python -m tools.naiz_img.inject --game <game> --yes` |
| `makegame.sh test <game>` | 启动测试 | `python tools/env_setup/install_env.py test-hdi --hdi disks/<game>.hdi` |
| `makegame.sh build <proj>` | 编译项目 | `make -C projects/<proj>` |
| `makegame.sh` (无参) | 交互菜单 | 列 games/ → 选操作 |

### 3.2 make 子命令

```bash
make)
    GAME="$2"
    if [ -z "$GAME" ]; then
        echo "用法: makegame.sh make <game>"
        exit 1
    fi
    shift 2
    exec python -m tools.naiz_img.inject --game "$GAME" --yes
    ;;
```

去掉 `make_hdi.sh` 原有的交互选择逻辑（用户已用 `--yes` 确认），简化。无参交互模式下才需要选择。

### 3.3 test 子命令

```bash
test)
    GAME="$2"
    if [ -z "$GAME" ]; then
        echo "用法: makegame.sh test <game>"
        exit 1
    fi
    shift 2
    exec "$PYTHON" tools/env_setup/install_env.py test-hdi --hdi "disks/$GAME.hdi"
    ;;
```

### 3.4 build 子命令

```bash
build)
    PROJ="$2"
    if [ -z "$PROJ" ]; then
        echo "用法: makegame.sh build <project>"
        exit 1
    fi
    shift 2
    if [ ! -d "projects/$PROJ" ]; then
        echo "错误: 项目不存在: projects/$PROJ"
        exit 1
    fi
    exec make -C "projects/$PROJ"
    ;;
```

### 3.5 无参交互模式

```bash
"")
    # 检测 games/ 下的游戏目录
    GAMES=()
    while IFS= read -r -d '' game; do
        GAMES+=("$(basename "$game")")
    done < <(find "$ROOT/games" -mindepth 1 -maxdepth 1 -type d -print0 | sort -z)

    if [ ${#GAMES[@]} -eq 0 ]; then
        echo "错误: games/ 下没有游戏"
        exit 1
    fi

    echo "===== 游戏工作流 ====="
    echo "检测到以下游戏:"
    for i in "${!GAMES[@]}"; do
        echo "  $((i+1))) ${GAMES[$i]}"
    done
    read -p "请选择游戏 [1]: " choice
    choice=${choice:-1}
    GAME="${GAMES[$((choice-1))]}"

    echo ""
    echo "操作选择:"
    echo "  1) build  — 编译项目数据"
    echo "  2) make   — 制作 HDI 镜像"
    echo "  3) test   — 启动 NP2kai 测试"
    echo "  4) all    — build → make → test"
    echo "  0) 返回"
    read -p "请选择 [0-4]: " action

    case $action in
        1) $0 build "$GAME" ;;
        2) $0 make "$GAME" ;;
        3) $0 test "$GAME" ;;
        4) $0 build "$GAME" && $0 make "$GAME" && $0 test "$GAME" ;;
        0) exit 0 ;;
        *) echo "无效选项" ;;
    esac
    ;;
```

### 3.6 venv 确保
