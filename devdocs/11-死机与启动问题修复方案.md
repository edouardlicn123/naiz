# 系统死机与启动问题修复方案

## 1. 现状与根因纠正

文档 10 将宿主机死归因于「SDL Wayland 输入抓取」，但实际运行环境是 **X11**（`XDG_SESSION_TYPE=x11`），该分析不成立。当前实际有两个独立问题：

### 问题 A：模拟器黑屏，进不了 DOS

| 原因 | 说明 |
|------|------|
| 缺少 `ROM.ROM` | `tools/pc98bios/` 只有 `font.rom`，无 BIOS ROM 文件。NP2kai 完成 POST 后无法做磁盘 I/O，黑屏挂死 |
| VBR 签名缺失 | `hdi_img.py:130-131` 将分区 VBR 的 `0x55AA` 签名清零，IPL 无法确认 VBR 合法 |

### 问题 B：宿主机系统死机

| 原因 | 说明 |
|------|------|
| NP2kai 无 BIOS 无法引导 → 陷入 100% CPU 死循环 | 整个系统响应卡死，鼠标键盘看似失效 |
| 无进程优先级控制 | NP2kai 与系统抢占 CPU，加剧卡顿感 |

文档 10 的引擎超时 + `timeout 30` 修复已就位，但问题 A 未解决时引擎根本跑不到，这些防护无效。

## 2. 修复步骤

### 2.1 安装 BIOS ROM 文件

`tools/BIOS.ROM`（96KB，来源：retrobios）→ 复制到 `tools/pc98bios/ROM.ROM`。

NP2kai 配置中 `biospath = tools/pc98bios`，会在此目录搜索 `ROM.ROM`。

### 2.2 修复 VBR 引导签名

**文件：** `tools/hdi_tool/hdi_img.py:130-131`

当前：
```python
vbr[0x1FE:0x200] = b'\x00\x00'
```

改为：
```python
vbr[0x1FE:0x200] = b'\x55\xAA'
```

PC-98 的 IPL 需要 VBR 的 `0x55AA` 签名才能确认其为合法引导扇区。

### 2.3 建立 MS-DOS 6.2 软链接

`install_dos.py` 期望路径 `[PC98] MS-DOS6.2/` 在项目根目录，实际在 `/home/edo/` 下。

创建软链接：
```bash
ln -s /home/edo/\[PC98\]\ MS-DOS6.2 /home/edo/naiz/
```

### 2.4 重建 DOS62.hdi

```bash
python3 tools/hdi_tool/install_dos.py
```
重新创建含 DOS 6.2 系统文件（IO.SYS / MSDOS.SYS / COMMAND.COM）的 80MB HDI 镜像。

### 2.5 加固 testtemp.sh 死机防护

| 措施 | 说明 |
|------|------|
| `nice -n 10` | 降低 NP2kai 进程优先级，不影响系统响应 |
| `timeout 15` | 从 30 秒缩短到 15 秒，更快自动恢复 |
| `SDL_VIDEO_ALLOW_SCREENSAVER=1` | 防止 SDL 独占输入设备 |

**文件：** `testtemp.sh`

```bash
export XDG_CONFIG_HOME=core
export SDL_VIDEO_ALLOW_SCREENSAVER=1
nice -n 10 timeout 15 sdlnp21kai_sdl2
```

### 2.6 其他文件不变

| 文件 | 状态 | 说明 |
|------|------|------|
| `core/engine/main.c` | ✅ 已有超时退出 (300帧) | 文档 10 已修复 |
| `core/plat/pc98/isr.asm` | ✅ 已有 `xor al, al` | 文档 10 已修复 |
| `core/sdlnp21kai/np21kai.cfg` | 不变 | 路径配置正确，只需 `ROM.ROM` 到位 |

## 3. 预期效果

1. `./testtemp.sh` 启动
2. NP2kai POST → 加载 BIOS ROM → 从 DOS62.hdi 引导 DOS
3. DOS 提示符 `A:\>` → 输入 `C:` → `ENGINE`
4. 引擎蓝色画面 → 5 秒自动退出回到 DOS
5. 15 秒后 `timeout` 兜底关闭模拟器
6. 整个过程 X11 不卡死，鼠标键盘始终可用

## 4. 验证方法

```bash
make -C core all && \
cp core/engine.exe core/engine.com && \
python3 tools/hdi_tool/hdi_inject.py tools/DOS62.hdi core/engine.com && \
rm -f core/engine.com
export XDG_CONFIG_HOME=core
nice -n 10 timeout 15 sdlnp21kai_sdl2
```
