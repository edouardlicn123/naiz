# 场景引擎

**源文件**: `src/sceneengine.c` (1198 行), `src/sceneengine.h` (60 行)

场景引擎是 MHVN98 的核心——一个自定义的字节码虚拟机 (VM)，用于驱动视觉小说的叙事逻辑。

## 数据结构

### 全局状态

```c
typedef struct {
    unsigned short numScenes;   // 总场景数
    unsigned short numChars;    // 总角色数
    unsigned short curScene;    // 当前场景编号 (0xFFFF = 未加载)
} SceneInfo;

SceneInfo sceneInfo;
unsigned char curSceneData[4096];   // 当前场景的解压字节码缓冲区
unsigned short curSceneDataPC;      // 程序计数器
unsigned char vmFlags;              // VM 状态标志
```

### VM 标志

```c
#define VMFLAG_Z         0x01  // 零标志（比较/算术结果：等于 0）
#define VMFLAG_N         0x02  // 负数标志（比较/算术结果：小于 0）
#define VMFLAG_TEXTINBOX 0x40  // 文本框中有文本
#define VMFLAG_PROCESS   0x80  // VM 是否继续执行
```

### 状态变量地址空间

```c
#define SVAR_BASE    0x0000   // 临时变量 (8 个)
#define SFLG_BASE    0x0008   // 临时标志 (24 个)
#define GVAR_BASE    0x0020   // 全局变量 (224 个)
#define GFLG_BASE    0x0100   // 全局标志 (768 个)
#define LVAR_BASE    0x0400   // 局部变量 (512 个)
#define LFLG_BASE    0x0600   // 局部标志 (2560 个)
#define VARSPACE_TOP 0x1000
```

| 区域 | 地址范围 | 数量 | 类型 | 范围 |
|------|----------|------|------|------|
| 临时变量 | `0x0000-0x0007` | 8 | `short` (16-bit signed) | 任意 |
| 临时标志 | `0x0008-0x001F` | 24 | 1-bit | 可用作 r0-r7/f0-f95 |
| 全局变量 | `0x0020-0x00FF` | 224 | `short` | 跨存档共享 |
| 全局标志 | `0x0100-0x03FF` | 768 | 1-bit | 跨存档共享 |
| 局部变量 | `0x0400-0x05FF` | 512 | `short` | 每个存档独立 |
| 局部标志 | `0x0600-0x0FFF` | 2560 | 1-bit | 每个存档独立 |

## 场景加载流程 `LoadNewScene()` (行 115–156)

```
OpenFile(sceneDataPath)
SeekFile → 读取场景指针表
SeekFile → 读取压缩场景数据
MemAlloc → LZ4Decompress → 解压到 curSceneData[4096]
LoadSceneText → 加载当前场景的文本数据
重置 PC、async 状态、VM 标志
```

## 场景初始化 `SetupSceneEngine()` (行 158–176)

```
OpenFile(SCENE.DAT)
读取 numScenes 和 numChars
申请 sceneTextBuffer (64KB)
调用 LoadNewScene(0) 加载第一个场景
```

## 主处理循环 `SceneDataProcess()` (行 558–1197)

被主循环每帧调用一次。核心逻辑：

```c
while (vmFlags & VMFLAG_PROCESS) {
    curOpcode = curSceneData[curSceneDataPC++];
    switch (curOpcode) {
        // 处理每个 opcode
    }
}
return returnStatus;
```

当 `VMFLAG_PROCESS` 被清除时（如遇到等待用户输入的 `text` 指令），循环暂停执行。

## 异步动作处理 `SceneAsyncActionProcess()` (行 431–556)

每帧在主循环中调用，处理三种类型的异步动画：

### 1. 淡入淡出动作 (`ASYNC_FADE`)
- `AFADE_BFADEIN/BFADEOUT` — 黑色淡入/淡出
- `AFADE_WFADEIN/WFADEOUT` — 白色淡入/淡出
- `AFADE_PHUEROTATE` — 色调连续旋转

使用 10.6 定点数跟踪当前淡入程度，每帧更新显示调色板。

### 2. 调色板混合 (`ASYNC_PALETTE`)
- `APAL_PFADEIN/PFADEOUT` — 在 main 和 mix 调色板之间渐变

### 3. 画面滚动 (`ASYNC_SCROLL`)
- `ASCR_SHAKE` — 垂直屏幕震动

使用三角函数生成正弦波位移，每帧衰减振幅。当振幅小于阈值 (`0x0080`) 时自动停止。

### `nowait` 机制

当一条指令后面紧跟着 `nowait` (0x0F) 操作码时，VM 不暂停执行（异步动作在后台继续）。否则 VM 暂停直到异步动作完成。

## 用户等待

`ASYNC_USER` 标志 (`0x80`) 用于标记需要用户输入才能继续的状态。通过 `EndUserWait()` 清除。

## 控制流控制

```c
void ControlProcess(unsigned char process);  // process=1 继续执行, process=0 暂停
```

用于：
- `text`: 暂停，等待用户按 Enter
- `choice`: 暂停，等待用户选择
- `fade/shake`: 暂停，等待动画完成（除非有 `nowait`）
