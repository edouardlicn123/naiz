# 键盘输入系统

**源文件**: `src/platform/pc98_keyboard.c/h`

## 键盘状态跟踪

MHVN98 通过直接读取 PC-98 键盘 BIOS 的数据区来获取键盘状态，不依赖 `INT 09h` 或 `INT 16h`的中断处理。

### 键盘缓冲区

```c
#define KEYBUF_SEG  0x0058   // 键盘 BIOS 数据段
#define KEYBUF_OFF  0x0012   // 键盘缓冲区偏移
```

`keybuf` 是一个 32 字节的数组（`pc98_keyboard.h:12`），存放当前所有按键的按下状态。每个位对应一个按键。

### 按键状态检测

```c
// 检测按键是否当前被按下
#define key_is_down(k)   (keybuf[(k) / 8] & (0x01 << ((k) % 8)))

// 检测按键是否刚刚被按下（上升沿）
#define key_pressed(k)   (key_is_down(k) && !prev_keybuf[(k) / 8] & (0x01 << ((k) % 8)))

// 检测按键是否刚刚释放（下降沿）
#define key_released(k)  (!key_is_down(k) && prev_keybuf[(k) / 8] & (0x01 << ((k) % 8)))

// 更新上一帧状态
void UpdatePrevKeyStatus(void);  // 复制 keybuf → prev_keybuf
```

`key_pressed` 是主循环中使用的主要函数，用于检测"刚按下"的边缘触发事件，避免同一次按键被重复处理。

### 扫描码定义

`pc98_keyboard.h` 定义了所有 PC-98 键盘按键的索引常量：

```c
#define K_0       0x00     // 数字键
#define K_1       0x01
// ...
#define K_A       0x20     // 字母键
// ...
#define K_ENTER   0x1D     // 回车
#define K_SPACE   0x3A     // 空格
#define K_ESC     0x01     // Escape
#define K_UP      0x49     // 方向键 ↑
#define K_DOWN    0x51     // 方向键 ↓
#define K_LEFT    0x4B     // 方向键 ←
#define K_RIGHT   0x4D     // 方向键 →
#define K_INS     0x52     // Insert
#define K_DEL     0x53     // Delete
#define K_HOME    0x47     // Home
#define K_END     0x4F     // End
#define K_PGUP    0x49     // Page Up (与 ↑ 共享)
#define K_PGDN    0x51     // Page Down (与 ↓ 共享)
// Shift (左/右): 0x2A, 0x36
// Ctrl: 0x1D
// GRPH: 0x38
// 等
```

## 主循环中的键盘处理

在 `main.c:223-260` 中：

| 按键 | 场景状态 | 行为 |
|------|----------|------|
| Enter | 选项选择中 | `CommitChoice()` → 提交选择 |
| Enter | 文本动画完成 | 推进下一段文本 |
| Enter | 文本动画中 | `textSkip=1` 跳过动画 |
| ↑ | 选项选择中 | `SwitchChoice(-1)` 向上选择 |
| ↓ | 选项选择中 | `SwitchChoice(1)` 向下选择 |
| Esc | 任意 | 退出引擎 |

## 更新频率

键盘状态在每个 VSync 中断（约 56.4Hz）时读取一次，调用 `UpdatePrevKeyStatus()` 保存上一帧快照以支持边缘检测。
