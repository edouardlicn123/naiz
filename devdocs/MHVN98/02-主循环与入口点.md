# 主循环与入口点

**源文件**: `src/main.c` (285 行)

## `CheckCompatibility()` — 平台检测 (行 44–106)

引擎启动时执行两项检测：

### 1. PC-98 平台检测

```c
__asm volatile (
    "mov $0x1000, %%ax\n\t"
    "int $0x1A\n\t"        // PC-98 打印机 BIOS 向量
    "jc .buggyIBM%=\n\t"   // 兼容 IBM BIOS 设置 carry 的情况
    "mov %%ax, %0\n\t"
    ".buggyIBM%=:"
);
```
- 调用 `INT 1Ah`（PC-98 的打印机 BIOS，IBM 兼容机上通常无效）
- 如果返回 `AX != 0x1000` 则认为是 PC-98
- 否则退出（"Unsupported platform!"）

### 2. EGC 硬件检测

在 VRAM 写入已知值，然后使用 EGC 的 XOR 功能验证硬件是否正确响应：

```c
EGCSetReadWriteMode(EGC_WRITE_ROPSHIFT | EGC_SOURCE_CPU | EGC_ROP(EGC_ROP_SRC ^ EGC_ROP_DST));
*((volatile unsigned __far short*)GDC_PLANES) = 0xFEED;
// 验证: p0 = 0xAAAA ^ 0xFEED, 等
```

### 主循环 (行 194–261)

```
while (1) {
    while (!vsynced) { __asm("hlt"); }  // 等待 VSync

    DoDrawRequests();                    // 执行待处理的绘制请求
    sceneProcessResult = SceneDataProcess();  // 处理场景字节码
    SceneAsyncActionProcess();           // 更新异步动作（淡入淡出/震动）

    if (ERROR)    → 显示错误，退出
    if (FINALEND) → 正常退出
    else if (WIPETEXT)      → ControlProcess(1)
    else if (RENDERTEXT)    → StringWriteAnimationFrame() 逐字动画
    else                    → 空闲

    vsynced = 0;

    // 键盘处理
    if (Enter) {
        if (选项选择中)  → CommitChoice(); EndUserWait()
        else if (动画完成) → 推进文本; EndUserWait()
        else               → textSkip = 1 (跳过动画)
    }
    if (↑) → SwitchChoice(SELECT_UP)    // 选择向上
    if (↓) → SwitchChoice(SELECT_DOWN)  // 选择向下
    if (Esc) → break                     // 退出

    UpdatePrevKeyStatus();
}
```

### 主循环返回值状态

定义于 `sceneengine.h`：

| 标志 | 值 | 含义 |
|------|-----|------|
| `SCENE_STATUS_RENDERTEXT` | `0x0001` | 正在逐字显示文本 |
| `SCENE_STATUS_WIPETEXT` | `0x0002` | 正在擦除文本 |
| `SCENE_STATUS_MAKING_CHOICE` | `0x0004` | 等待用户选择 |
| `SCENE_STATUS_FINALEND` | `0x4000` | 故事结束 |
| `SCENE_STATUS_ERROR` | `0x8000` | 发生错误 |

## 中断管理

- 使用 `INT 1Ah` 检测平台
- 安装 VSync 中断处理（用 `Isr.asm` 中的 `VsyncInterrupt` 替代默认向量）
- 通过 8259A PIC 控制中断掩码
