## 图像管理

### 核心数据结构 — `ImageInfo`

```c
typedef struct imginf {
    Rect2Int boundRect;                  // 图像的边界矩形
    __far unsigned char* mask;           // 遮罩平面（可选）
    __far unsigned char* plane0;         // 4 个颜色平面
    __far unsigned char* plane1;
    __far unsigned char* plane2;
    __far unsigned char* plane3;
    struct imginf* children;             // 子图像链表（用于 9-slice）
    unsigned short id;                   // 唯一标识符
    unsigned char layer;                 // 层号
    unsigned char flags;                 // 状态标志
} ImageInfo;
```

### 图像标志

```c
#define IMAGE_TYPE_NORMAL 0x00   // 普通图像
#define IMAGE_TYPE_9SLICE 0x01   // 9-slice 图像
#define IMAGE_MEM_NORMAL  0x00   // 内存存储
#define IMAGE_MEM_VRAM    0x02   // VRAM 存储
#define IMAGE_ALIGN_FREE  0x00   // 自由对齐
#define IMAGE_ALIGN_FIXED 0x04   // 固定对齐
#define IMAGE_LOADED      0x80   // 已加载
#define IMAGE_DRAWN       0x40   // 已绘制
#define IMAGE_DRAWREQ     0x20   // 有待处理绘制请求
```

### 绘制请求队列

图形系统使用基于标志的请求机制：

1. 设置 `ImageInfo.flags |= IMAGE_DRAWREQ` 标记需要重绘
2. 主循环调用 `DoDrawRequests()` 处理所有标记的图像
3. 处理时绘制后清除 `IMAGE_DRAWREQ` 标志

```c
void DoDrawRequests();    // 处理所有绘制请求
void RedrawEverything();  // 全屏重绘
```

## GPI 图像格式

**GPI** (General Planar Image) 是 MHVN 引擎使用的自定义图像格式。

### 文件头 (14 字节)

```c
#define GPI_HEADER_SIZE 0x0E

0x00  char[3]   magic           "GPI"
0x03  uint8     flags           压缩/字节序/BPC 标志
0x04  uint16    width - 1       图像宽度 (像素) − 1
0x06  uint16    height - 1      图像高度 (像素) − 1
0x08  uint16    numTiles - 1    瓦片数 − 1
0x0A  uint16    planeMask       位面掩码 (bit 0-3 = 颜色面, bit 8 = 遮罩面)
0x0C  uint16    planeFilterMask 每个位面的滤波器类型
```

### `OpenGPIFile()` (gpimage.c:34-89)

1. 验证魔数 "GPI"
2. 解析宽度、高度、瓦片数
3. 根据 `planeMask` 分配颜色平面和遮罩平面内存
4. 记录 `planeFilterMask` 用于后续反滤波器

### `DecompressGPIFile()` (gpimage.c:92-283)

对每个位面：
1. 如果被滤波，读取滤波规范（每行 4-bit 滤波器类型）
2. 读取 LZ4 压缩块 → 解压
3. 如果被滤波，执行反滤波：

| 滤波器 ID | 操作 |
|-----------|------|
| 0x0 | 无操作 |
| 0x1 | XOR 游程编码（逐行像素差分） |
| 0x2 | XOR 上行 |
| 0x3 | XOR 上一瓦片同位置行 |
| 0x4-0x7 | XOR 同位置的 plane 1-4 数据 |
| 0x8 | 取反 |
| 0x9 | 取反 + 游程编码 |
| 0xA-0xE | 取反 + XOR 对应参考行/面 |
| 0xF | XOR ~参考面 |

最后，遮罩面取反（使 1=不透明，0=透明）。

## 9-Slice 边框系统

### 默认边框定义 (graphics.c)

内置的 9-slice 边框数据使用 16×16 像素的瓦片，在 4 个颜色平面上分别定义：

```c
// 四个角、四条边、中心区域 共 9 个瓦片
// 左上、上、右上、左、中、右、左下、下、右下
```

ImageInfo* RegisterTextBox(const Rect2Int* rect);      // 注册文本框

### 加载到 VRAM

```c

```

## 背景图像加载

```c
ImageInfo* LoadBGImage(unsigned int num);
```

1. 打开 `BGIMAGE.DAT`
2. 从指针表读取第 `num` 个背景的数据指针
3. 解析背景结构（主图指针、调色板索引、变体数量）
4. 读取主背景 GPI 数据，解码到 `ImageInfo`
5. 设置调色板为背景对应的调色板

## 图像卸载

```c
void UnloadImage(ImageInfo* img);
```

释放图像的所有显存和内存资源。

## `RedrawEverything()` 调用链

当 `setbg` 指令执行时：
```
setbg(0x30) → LoadBGImage(bgNum) → ImageInfo.flags |= IMAGE_DRAWREQ
→ RedrawEverything() → 标记所有已注册图像为 DRAWREQ
→ 下一帧 DoDrawRequests() 执行实际绘制
```
