# 文本引擎

**源文件**: `src/textengine.c` (1243 行), `src/textengine.h` (82 行)

文本引擎负责所有文字的渲染工作，包括逐字动画输出、富文本格式化和自动换行。

## 整体流程

```
场景引擎调用 text(0x11)
    │
    ▼
StartAnimatedStringToWrite(str, x, y, format)
    │
    ├─ PreprocessString() — 预处理（注入变量、自动换行）
    ├─ PreloadGlyphs()    — 预加载所有 glyph 到缓存
    └─ 初始化动画状态
    │
    ▼
主循环每帧调用 StringWriteAnimationFrame(skip)
    │
    ├─ skip=1 → 直接输出全文
    └─ skip=0 → 逐字动画输出
```

## 文本预处理 `PreprocessString()` (行 659–788)

### 变量注入

处理 `0x1B 0x5x` 转义序列（`@p` 命令），将 `customInfos[x]` 字符串插入当前位置。这是一种类似 `sprintf` 的机制，用于插入主角姓名等动态内容。

### 自动换行

扫描文本中的每个字符，计算其宽度，在超出文本框右边界时自动插入 `CR+LF`（`0x0D 0x0A`）。

换行逻辑：
- 遇到空格 (`0x20`) 记录断点
- 遇到连字符隐含断点（代码中被注释掉的逻辑）
- 超过右边界时回退到最近断点插入换行
- 无合适断点时紧急换行

## 格式化系统

格式编码在 `short format` 变量中：

```
bit  3     2     1     0
    ┌─────┬─────┬─────┬─────┐
    │ 颜色│ 淡入│      │ SIB │
    └─────┴─────┴─────┴─────┘
    B=粗体, I=斜体, U=下划线, S=阴影
```

```c
#define FORMAT_BOLD      0x0001
#define FORMAT_ITALIC    0x0002
#define FORMAT_UNDERLINE 0x0004
#define FORMAT_SHADOW    0x0008
#define FORMAT_FADE(n)   (((n) << 8) & 0x0F00)
#define FORMAT_COLOUR(n) (((n) << 12) & 0xF000)
```

### 转义序列格式

文本中的格式化通过 `0x1B` (ESC) 后跟控制字节实现：

| 字节范围 | 功能 |
|----------|------|
| `0x1x` | 设置格式标志 (x=SUIB: 阴影/下划线/斜体/粗体) |
| `0x3x` | 设置淡入级别 (x=0~F) |
| `0x4x` | 设置颜色索引 (x=0~F) |
| `0x5x` | 主角信息注入 (x=slot 编号) |
| `0x6x` | 等待 10×(x+1) 帧 |
| `0x7x` | 设置字符间等待时间为 (x+1) 帧 |
| `0x8x` | 插入变量值（带地址高位） |
| `0xAx` | 根据标志切换文本（带地址高位） |
| `0xFx` | 重置格式至默认 (x=CFTM) |

## 逐字动画 `StringWriteAnimationFrame()` (行 997–1236)

### 缓冲区结构

```c
unsigned long animCharBuf[16 * 16];  // 环形缓冲区，最多 16 个字符
short charXs[16], charYs[16];        // 每个字符的位置
char charColours[16];                 // 每个字符的颜色
char charFlags[16];                   // 阴影/32px 标志
unsigned char charFade[16];          // 每个字符的淡入进度
```

### 动画流程

1. **输入新字符**（如果等待帧数为 0）：
   - 解码下一个 UTF-8 字符
   - 加载 glyph 数据到缓冲区
   - 应用格式效果（粗体、斜体等）
   - 设置位置、颜色、初始化淡入度为 0
   - 如果是空格、换行等控制字符，直接更新位置

2. **绘制**：
   - 遍历缓冲区中所有字符（最多 16 个）
   - 每个字符的淡入进度递增（`charFade[n]++`, 范围 0-16）
   - 使用 Bayer 4×4 抖动表 (`bayer4x4masks`) 控制淡入可见度
   - 先绘制阴影（如果启用），再绘制前景

3. **跳过**：
   - 当用户按 Enter 时设 `skip=1`
   - 直接使用 `WriteStringInternal()` 全文输出

### Bayer 抖动表

```c
const unsigned short bayer4x4masks[64] = {
    // 整个 4×4 Bayer 矩阵的 16 级灰度值
    // 每个值重复 4 次以填充 64 个条目
    0x8888, 0x0000, 0x0000, 0x0000,  // 级别 0
    0x8888, 0x0000, 0x2222, 0x0000,  // 级别 1
    // ...
    0xFFFF, 0xFFFF, 0xFFFF, 0xFFFF   // 级别 15
};
```

## 即时文本输出 `WriteStringInternal()` (行 822–995)

与动画版本不同，`WriteStringInternal` 立即渲染整个字符串，适用于：
- 角色名显示
- 选项菜单
- 跳过动画时的全文输出

## 字符绘制

### 格式应用顺序

```
原始 glyph → ItaliciseChar → BoldenCharRight → UnderlineChar → MaskChar(淡入)
→ SwapCharDataFormats(转为 VRAM 兼容格式) → DrawChar(输出到显存)
```

### `DrawChar()` (行 303–425)

使用 EGC 将 glyph 数据写入 PC-98 显存：
- 支持 8 位宽 (16 位/行) 和 16 位宽 (32 位/行) 字形
- 处理未对齐（非 8 像素边界）的情况
- 使用 EGC 位地址转移功能实现亚像素级位移
- 手写内联汇编（展开循环以减小开销）

```c
unsigned short* planeptr = (unsigned short*)(y * 80 + ((x >> 3) & 0xFFFE));
EGCSetBitAddressTransferDirection(EGC_BLOCKTRANSFER_FORWARD | EGC_BITADDRESS_DEST(xinblock));
// 未对齐时需要额外的 dummy 写入来清空 EGC 移位缓冲区
```

### `DrawCharMask()` (行 428–652)

带 Bayer 抖动的版本，用于逐字淡入效果：
- 每行写入时先与抖动掩码进行 `AND` 运算
- 同样使用展开的内联汇编

## 阴影颜色

```c
const unsigned char stdShadowCols[16] = {
    0x0, 0x0, 0x0, 0x2, 0x0, 0x4, 0x5, 0x0,
    0x0, 0x8, 0x9, 0x7, 0x0, 0xC, 0xD, 0x1
};
```

每个文本颜色对应的阴影颜色映射，阴影像素绘制在原字符右下偏移 (1,1) 像素处。

## 文本框管理

引擎维护三个 UI 元素：

```c
Rect2Int textBoxInnerBounds;      // 文本显示区域 (80,288, 480×64)
ImageInfo* textBoxImgInfo;        // 文本框 9-slice 图像

Rect2Int charNameBoxInnerBounds;  // 角色名区域 (80,240, 160×16)
ImageInfo* charNameBoxImgInfo;

Rect2Int choiceBoxInnerBounds;    // 选择框区域 (336,160, 256×64)
ImageInfo* choiceBoxImgInfo;
```

每个 UI 元素使用 9-slice 边框渲染（`Draw9SliceBoxInnerRegion()`）。
